# 开发环境配置

### 元工具：West

```bash
# 安装 west
uv tool install west

# 初始化工作空间后
# 安装 Python 依赖包
cd ~/zephyr_workspace/zephyr
uv pip install -r scripts/requirements.txt
```

### 初始化工作空间：

```bash
# 创建工作目录
mkdir zephyr_workspace && cd zephyr_workspace

# 初始化 west 工作区（使用 Zephyr 主分支）
west init -m https://github.com/zephyrproject-rtos/zephyr

# 更新所有模块（这一步会下载 Zephyr 及其依赖）, 时间很长
west update

# 导出 Zephyr CMake 包（让 CMake 能找到 Zephyr）
west zephyr-export
```

### 安装 Zephyr SDK：

```bash
# 下载最新版本sdk并解压
cd zephyr-sdk-x.xx.x
.\setup.cmd
```

`\zephyr-sdk-1.0.1\hosttools\openocd` 自带openocd有问题, 软链接到能用的, 比如`xpack-openocd-x.xx.x`

### TI MSPm0g3507编译烧录验证：

```bash
# 进入 Zephyr 目录
cd ~/zephyr_workspace/zephyr

# 编译示例, pwm控制LED闪烁频率, 第一次巨慢
west build -p always -b lp_mspm0g3507 samples/basic/blinky_pwmd

# 烧录, 记得reset
west flash
```



# West 核心命令

### west build - 构建项目

**命令格式**：

```
west build -b BOARD [-d BUILD_DIR] [SOURCE_DIR] [-- CMAKE_ARGS]
```

**构建选项表格**：

| 选项           | 说明                       | 示例                     |
| :------------- | :------------------------- | :----------------------- |
| `-b BOARD`     | 指定目标板子               | `-b nrf52840dk_nrf52840` |
| `-d BUILD_DIR` | 指定构建目录               | `-d build`               |
| `-p auto`      | 自动清理（pristine build） | `-p auto`                |
| `-p always`    | 总是清理重建               | `-p always`              |
| `-t TARGET`    | 构建特定目标               | `-t menuconfig`          |
| `--cmake-only` | 只运行 CMake，不编译       | `--cmake-only`           |
| `-c`           | 增量构建（默认）           | `-c`                     |

**常用示例**：

`west build -b lp_mspm0g3507 -d build samples/hello_world -p auto`

### west flash - 烧录固件

**命令格式**：

```bash
west flash [-d BUILD_DIR] [--runner RUNNER] [--context] [RUNNER_ARGS]
```

### 其他命令

**`west debug` - 调试程序 (冷启动)**

**`west attach` - 现场调试 (运行到一半接管) ** 



# 工程结构

典型项目结构

```
my_zephyr_app/
├── CMakeLists.txt          # CMake 构建配置文件
├── prj.conf                # 项目配置文件
├── src/                    # 源代码目录
│   ├── main.c              # 主程序入口
│   └── ...                 # 其他源文件
├── boards/                 # 板级配置目录（可选）
│   └── nrf52840dk_nrf52840.overlay  # 设备树 Overlay
├── include/                # 头文件目录（可选）
│   └── ...
├── lib/                    # 自定义库目录（可选）
│   └── ...
└── README.md               # 项目说明文档
```

### CMakeLists

```cmake
# 查找 Zephyr 包（必须在 project() 之前）
find_package(Zephyr REQUIRED HINTS $ENV{ZEPHYR_BASE})
```

### 项目级Kconfig (可选)

自定义参数字典，可在根目录（项目级） / 每个自定义驱动模块下添加

* bool型：逻辑功能开关，一般在 prj.conf 中启用
* 其他：具体参数，一般在 .dts 板级配置中规定

### prj.conf

* 配置软件行为：启用组件功能
  * 官方底层驱动GPIO / 文件系统 / 日志等
  * 自定义Kconfig逻辑功能开关

### boards/板级配置

在.dts(默认硬件配置) 上 叠加.overlay (在原始板级文件(通常全disable)上修改硬件描述，"注册"需要使用的硬件)

* 用于描述硬件特性，包括Kconfig参数
  * 官方底层驱动：GPIO 等
  * 自定义驱动：电机等

* 使用官方适配的板子：`boards/`下添加`board_name.overlay`，描述需要的硬件节点

* 使用第三方板子：

  参考官方适配好的板子：`zephyr/boards/<arch>/<board_name>/`

  ```
  my_custom_project/
  ├── boards/
  │   └── <vendor>/
  │       └── <board_name>/
  │           ├── <board_name>.dts         # 设备树定义
  │           ├── <board_name>_defconfig   # 默认内核配置
  │           ├── board.yml                 # 板子元数据 (Zephyr v3.7+)
  │           ├── Kconfig                   # Kconfig 定义
  │           ├── Kconfig.defconfig         # 默认 Kconfig 值
  │           └── CMakeLists.txt            # 板级构建脚本
  ├── CMakeLists.txt
  └── prj.conf
  ```
