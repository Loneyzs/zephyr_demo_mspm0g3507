# 工程结构

典型项目结构

```
my_zephyr_app/
├── CMakeLists.txt          # CMake 构建配置文件
├── prj.conf                # 项目配置文件
├── src/                    # 源代码目录
│   ├── main.c              # 主程序入口
│   └── ...                 # 其他源文件
├── boards/                 # 板级配置目录（可选）
│   └── nrf52840dk_nrf52840.overlay  # 设备树 Overlay
├── include/                # 头文件目录（可选）
│   └── ...
├── lib/                    # 自定义库目录（可选）
│   └── ...
└── README.md               # 项目说明文档
```



## CMakeLists

```cmake
# 查找 Zephyr 包（必须在 project() 之前）
find_package(Zephyr REQUIRED HINTS $ENV{ZEPHYR_BASE})
```



## 软件 Config 参数

Zephyr 使用 Kconfig 进行参数字典管理

#### 项目级Kconfig (可选)

自定义参数字典，可在根目录（项目级） / 每个自定义驱动模块下添加

* bool型：逻辑功能开关，一般在 prj.conf 中启用
* 其他：具体参数，一般在 .dts 板级配置中规定

#### prj.conf

* 配置软件行为：启用组件功能
  * 官方底层驱动GPIO / 文件系统 / 日志等
  * 自定义Kconfig逻辑功能开关



## boards/ 板级设备树配置

在 .dts(默认设备树) 上 叠加 .overlay 进行硬件描述

* 官方底层驱动：GPIO 等
* 自定义驱动：电机等也可写成设备树节点

#### 使用官方适配的开发板：

* 查看`zephyr/boards/<arch>/<board_name>/`下dts已添加的外设
* `boards/`下添加`board_name.overlay`，描述需要的硬件节点

#### 使用第三方开发板：

* 参考官方适配好的板子：`zephyr/boards/<arch>/<board_name>/`
* 按如下结构添加自定义开发板

```
my_custom_project/
├── boards/
│   └── <vendor>/
│       └── <board_name>/
│           ├── <board_name>.dts         # 设备树定义
│           ├── <board_name>_defconfig   # 默认内核配置
│           ├── board.yml                 # 板子元数据 (Zephyr v3.7+)
│           ├── Kconfig                   # Kconfig 定义
│           ├── Kconfig.defconfig         # 默认 Kconfig 值
│           └── CMakeLists.txt            # 板级构建脚本
├── CMakeLists.txt
└── prj.conf
```
